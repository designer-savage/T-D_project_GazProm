# Перспективы масштабирования

Архитектурные направления для перехода от прототипа к корпоративной платформе. Не планы к немедленной реализации — ориентиры для питч-дека и дорожной карты.

---

## Откуда берётся контекст

Анализ ведущих T&D-платформ: SAP SuccessFactors + Joule, Workday + Sana, Cornerstone Galaxy, Degreed Maestro, Docebo Harmony, Microsoft Viva, Oracle HCM.

---

## 1. Skill Gap Analysis

**Что даёт:** Агент карьеры переходит от общих рекомендаций к конкретным — «тебе не хватает именно этого навыка до следующего грейда».

**Как у лидеров:** Cornerstone (55k навыков) и Degreed строят весь пайплайн вокруг gap: текущий уровень → целевой уровень роли → конкретный курс для закрытия.

**Что потребуется:**
- Таблица `role_requirements` (grade, skill_name, required_level)
- JOIN в `career_agent.py` с вычислением реального gap
- Seed-данные с требованиями для junior/middle/senior/lead

---

## 2. Семантический поиск (улучшенный RAG)

**Что даёт:** «Командировочные расходы» находит документ с «расходы на служебные поездки». Сейчас FTS5 это не умеет.

**Как у лидеров:** Workday/Sana и Docebo Neural Search — поиск по смыслу, а не по словам.

**Что потребуется:**
- `chromadb` или `sqlite-vec` + `sentence-transformers` (~120 МБ, офлайн)
- `rag/embedder.py` — индексация `knowledge_documents` при старте
- Hybrid search: векторный (семантика) + FTS5 (fallback)

---

## 3. Team Readiness Dashboard

**Что даёт:** Менеджер видит команду целиком — кто готов к повышению, где командная слабость, динамика за квартал.

**Как у лидеров:** Cornerstone и Oracle дают карту готовности с прогресс-барами, флагами и бар-чартами по компетенциям.

**Что потребуется:**
- `GET /dashboard/team-readiness?manager_id=mgr_001`
- Компонент с таблицей: сотрудник / грейд / прогресс / KPI / статус
- Данные уже есть в БД — нужны SQL-запросы и UI

---

## 4. Карьерные треки с визуализацией

**Что даёт:** Сотрудник видит конкретную дорожную карту — что откроется на следующем грейде, какие роли, какие задачи.

**Как у лидеров:** SAP Opportunity Marketplace и Oracle Job Discovery Agent матчат внутренние вакансии с профилем сотрудника.

**Что потребуется:**
- Страница `/career` с динамическим треком вместо статики
- Визуализация: текущий грейд → следующий → через один, с gap-ами на каждом переходе
- Зависит от Skill Gap Analysis (п. 1)

---

## 5. ActionAgent — транзакционные действия

**Что даёт:** Система не только отвечает, но и действует — записывает на курс, обновляет компетенцию, оформляет заявку прямо из чата.

**Как у лидеров:** Workday + Sana — единый front door: ответ + транзакция в одном интерфейсе. Поиск по всей корпоративной базе (SharePoint, Slack, таблицы).

**Что потребуется:**
- `agents/action_agent.py` с детекцией транзакционных интентов
- Ответ агента расширяется: `{"text": "...", "action": {"type": "enroll_course", ...}}`
- SSE-событие `action_executed` — фронтенд обновляет UI без перезагрузки
- Обработка `action` в оркестраторе с реальным UPDATE в БД

---

## 6. Career Coach — симуляция рабочих сценариев

**Что даёт:** Обучение становится активным — сотрудник отрабатывает переговоры, конфликты, технические решения в безопасной среде с мгновенной обратной связью.

**Как у лидеров:** Docebo + Degreed AI-симуляторы с оценкой по шкале компетенций.

**Что потребуется:**
- `agents/coach.py` — агент-наставник, генерирует сценарий под роль/грейд, оценивает ответ
- Переключатель «Режим коучинга» на странице `/career`
- `mode=coaching` в запросе — оркестратор роутит напрямую к coach, минуя классификатор

---

## 7. Проактивная система уведомлений

**Что даёт:** Информация находит человека, не наоборот. Дедлайны, новые курсы по навыку, изменения в треке — всё приходит само.

**Как у лидеров:** Microsoft Viva и SAP Joule — персональный дайджест, умные напоминания, push при изменении статуса.

**Что потребуется:**
- Фоновый scheduler (APScheduler или Celery)
- Push-механизм (WebSocket или FCM)
- Проактивный агент, который мониторит изменения в `learning_progress` и `kpi_records`

---

## 8. Ролевая модель доступа

**Что даёт:** Чувствительные данные (KPI, оценки 360°, грейды) под контролем компании. Сотрудник видит своё, менеджер — команду, HR — всех.

**Сейчас:** `employee_id` хардкодом в теле запроса, переключение кнопкой в хедере.

**Что потребуется:**
- JWT-аутентификация или корпоративный SSO (Keycloak / Azure AD)
- Middleware проверки прав в `routers/`
- Логирование запросов к AI для аудита безопасности
