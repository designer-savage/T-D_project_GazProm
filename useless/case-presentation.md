# Кейс HSCC — ИТ-кластер «Газпром Нефти»

## Задача

**Создать корпоративную экосистему на базе универсальных AI-агентов для T&D ИТ-кластера «Газпром Нефти».**

---

## Шаги решения

1. Исследовать лучшие решения на рынке корпоративных платформ для обучения сотрудников и коммуникации в компании.
2. Проанализировать и сравнить функционал изученных платформ.
3. Подробно описать наполнение экосистемы, которая решает проблемы, изложенные в кейсе.
4. Визуализировать и наглядно представить основные экраны платформы.
5. Создать план разработки и внедрения портала (срок: не более 1 года).
6. Описать потенциальные риски проекта и способы их минимизации.
7. Рассчитать стоимость разработки и внедрения продукта.

---

## Боли, которые нужно решить

- **Библиотека курсов без приоритетов** — специалисты тратят время на выбор курсов, которые не соответствуют их текущим задачам.
- **Онбординг без ориентиров** — новые сотрудники проходят курсы последовательно, не понимая приоритетов. Руководитель не видит реального прогресса каждого.
- **Карьерные треки вручную** — менеджеры собирают данные из KPI, оценок 360°, грейдов из разных систем и строят траекторию разработки сотрудника вручную. Всё это можно автоматизировать.
- **Разрозненная коммуникация** — вопрос о занятости коллеги требует проверки почты, календаря и мессенджеров. Инструменты не унифицированы.
- **Нет единой среды** — нужна платформа с персональными рекомендациями по обучению, планированием совместной работы и защитой чувствительных данных.
- **Информация не находит человека** — система пассивна, сотрудник сам ищет всё нужное.

## Ключевые тезисы для защиты

1. **Данные не покидают контур компании** — LLM либо российская (GigaChat), либо self-hosted. Это принципиально для Газпром Нефти.
2. **Агенты независимы** — если один агент падает, остальные работают. Система устойчива.
3. **Система обучается** — чем больше сотрудников пользуются, тем лучше становятся рекомендации (feedback loop через логи взаимодействий).
4. **Интеграция без замены** — система не ломает существующие HR-инструменты, она читает из них данные через API.

## Целевая архитектура (для презентации)

```
Сотрудник
    → Единый портал (веб + Telegram-бот + мобильное приложение)
    → Оркестратор (LangGraph) → AI-агенты (LLM + RAG)
    → Skills Engine (граф навыков из Jira/GitLab/1С)
    → Learning Engine (персонализированные треки)
    → Manager Dashboard (карта готовности команды)
    → HR Analytics (ROI, succession, workforce planning)
```

**Стек:**

| Слой | Технологии |
|---|---|
| LLM | GigaChat Pro (данные в контуре РФ) / YandexGPT / self-hosted Llama |
| Оркестрация | LangGraph (граф состояний агентов) |
| Бэкенд | Python, FastAPI, LangChain |
| БД | PostgreSQL (профили, KPI, курсы) + Qdrant (векторная БД для RAG) + Redis (кэш) |
| Фронтенд | Next.js, React, TypeScript, Tailwind CSS |
| Инфраструктура | Docker, Kubernetes (прод), on-premise или частное облако |
| Интеграции | 1С:ЗУП, корпоративный SSO (Keycloak / AD), LMS компании |
